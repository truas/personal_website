## Goal
Make the personal website (GitHub Pages, plain static HTML/CSS/JS) more responsive, modern, and mobile-friendly. Default theme stays dark. Replace the slow Bibbase JSONP publications page with a fast, in-page renderer fed by a local JSON file. Keep all written content unchanged.

## What was done

### 1. Publications data pipeline (Bibbase → local JSON)
- Added **`tools/generate-publications.mjs`** — Node ≥18 script that:
  - fetches `https://bibbase.org/show?bib=…ruasterry/2503580&jsonp=1`,
  - unwraps the JSONP, extracts the embedded structured `data: [...]` array,
  - extracts the raw BibTeX from `<pre>` blocks,
  - pairs structured entries with BibTeX by index (Bibbase emits them in the same order),
  - normalises each entry into `{id, type, bibtype, year, title, authors[], venue, publisher, doi, url, abstract, bibtex}`,
  - sorts newest-first by year and writes `data/publications.json`.
- Generated **`data/publications.json`** — 77 entries, 2009–2026, 36 with abstracts, 61 with DOIs, all with BibTeX. ~196 KB.
- **Regeneration**: `node tools/generate-publications.mjs` from the repo root.

### 2. CSS refresh (`css/styles.css`)
Rewrote the stylesheet keeping the existing glassy navbar aesthetic. New design tokens, smoother colors, dark-mode bg darkened (`#0f1115`), elevated card surface (`--bg-elev`). New additions:
- Spacing & radius scale (`--s-1`…`--s-7`, `--radius-sm/md/lg`).
- `font-display`-friendly system stack + Inter as first choice.
- `:focus-visible` outlines, `prefers-reduced-motion` block disabling hover transforms.
- Theme-aware GitHub icon color (`--gh-icon`) so it's not invisible in dark mode.
- Profile image scales with `clamp()` (120 → 170 px).
- New breakpoints: 480 px (small phone polish) and 1100 px (slight container widening).
- New publications components: `.pubs-toolbar` (sticky), `.pubs-search`, `.chip`, `.pubs-year-group` (uses native `<details>`), `.pub-card`, `.pub-actions`, `.pub-btn`, `.pub-detail`, `.pub-bibtex` with `.pub-bibtex-copy`.

### 3. Publications page (`publications.html`)
Replaced the Bibbase `<script>` with a render scaffold: search input + two chip rows (Year, Type) + `<div id="publications-list">`. Loads `js/publications.js`.

### 4. Publications renderer (`js/publications.js`)
Vanilla JS, no deps. Loads `data/publications.json`, then:
- Search across title + authors + venue (case-insensitive, 120 ms debounce).
- Year chips + Type chips (Conference / Journal / Preprint / Workshop / Book / Report / Thesis / Other). Filters combine with AND.
- Groups results by year into native `<details open>` collapsibles with counts.
- Each card: title (link to URL if present), authors with last-name "Ruas" bolded, `venue · year · type`, action row with `DOI`, `Link`, `Abstract`, `BibTeX` (toggle reveals panel; Copy button writes to clipboard).
- Empty / loading / error states.
- Accessible: `aria-pressed`, `aria-expanded`, `aria-live` on stats.

### 5. Navbar / JS cleanup
- **`js/load-dynamic-content.js`** rewritten: default theme stays `dark`, deduped the second theme-apply block, sets `aria-current="page"` on the active link, closes mobile menu on link tap, toggles `aria-expanded` on the hamburger.
- **`navbar.html`**: added `aria-label`, `aria-expanded`, `aria-controls`, `width/height` on logo, decorative `alt=""`.

### 6. A11y & responsiveness on existing pages
- `index.html`, `about.html`: portrait image now has `width/height` (no CLS), `loading`/`decoding` attrs, descriptive alt; social links have `rel="noopener noreferrer"` and icons marked `aria-hidden`.
